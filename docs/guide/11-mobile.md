# 11 — Mobile (terrain)

> 📸 **Capture 1** : `/m` (accueil mobile) — accès aux chantiers du jour.
> 📸 **Capture 2** : `/m/site/[id]`. Annotations : ① nom du site + Nᵉ passage · ② **Démarrer une visite** · ③ bouton de **capture** (photo) · ④ **Aujourd'hui ici** (interventions / actions / anomalies) · ⑤ **Compte-rendu**.

## 🎯 Objectif
Capturer la mémoire **sur le chantier**, sans friction, et consulter l'essentiel.

## 🕒 Quand l'utiliser
Sur place, pendant une visite ou une intervention.

## 🔘 Les éléments clés
- **Démarrer une visite** — pose un repère ; capture libre pendant ; **Terminer** au départ.
- **Capture** — photo / note / vocal en un geste ; aucune saisie obligatoire.
- **Aujourd'hui ici** — ce qui me concerne maintenant (interventions du jour, actions, anomalies, dernières preuves).
- **Préparer ma visite** — le même briefing qu'au bureau, version mobile.
- **Compte-rendu** — saisie multimodale (voix + texte + photos).

## 🧭 Parcours conseillé
Ouvrir le site (via `/m` ou un QR) → Démarrer une visite → capturer → Terminer → débrief au bureau.

## 💡 Conseils
Le Nᵉ passage est une **signature**, pas un indicateur de performance. Mode pensé pour le terrain : 1 idée principale par écran, gros boutons.

## ⚠️ Erreurs fréquentes
La boîte verte « Visite en cours » signifie qu'une visite est ouverte : pensez à **Terminer** en repartant.
