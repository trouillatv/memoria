# 05 — Le PV

> 📸 **Capture 1** : `/meetings/[id]/pv/validation`. Annotations : ① parcours numéroté ①→⑥ · ② **aperçu vivant** du PDF à droite · ③ blocs éditables (participants, décisions, actions, photos).
> 📸 **Capture 2** : le bandeau **« Retrouver automatiquement ces points lors des prochaines réunions ? »** (bloc Décisions). Annotations : ① ✓ Déjà connu · ② ➕ Sera ajouté à la mémoire · ③ ⚠ Ressemble à une action · ④ bouton **Suivre**.

## 🎯 Objectif
Produire le **livrable opposable** : le procès-verbal.

## 🕒 Quand l'utiliser
Juste après la réunion, pour finaliser et diffuser.

## 🔘 Les éléments clés
- **Parcours numéroté** : ① Vérifier → ② Relire → ③ Corriger → ④ Générer → ⑤ Téléverser le final → ⑥ Figer la référence.
- **Aperçu vivant** — le vrai PDF à droite, rechargé à chaque modification.
- **Blocs éditables** — participants, casting (rôle→entreprise→contact), décisions, actions, photos (couverture, ordre, légendes).
- **Bandeau « Retrouver automatiquement ces points »** — un clic sur **Suivre** rattache décisions **et** actions liées à leurs **points suivis** (chap. 07).

## 🧭 Parcours conseillé
Corriger les blocs → **Générer** → **Téléverser la version finale diffusée** (= la preuve) → éventuellement **Figer** la version générée (= la référence MemorIA).

## 💡 Conseils
- Au bloc Décisions, **valider les points proposés en un clic** : la mémoire se construit sans ressaisie.
- Remplir le champ **Sujet** d'une décision fait apparaître la proposition de point.

## ⚠️ Erreurs fréquentes
- Confondre **version finale diffusée** (preuve, jamais écrasée) et **version générée figée** (référence interne, optionnelle).
- Décocher tous les points : on perd l'enrichissement automatique de la mémoire.
