# 06 — Actions

> 📸 **Capture 1** : `/actions` (transverse, tous chantiers). Annotations : ① **+ action** · ② recherche · ③ filtre statut (Ouvertes / Planifiées / Terminées) · ④ filtre corps d'état · ⑤ santé (🔴 critique / 🟠 surveiller / 🟢 rythme) · ⑥ sur une action : **Clôturer · Planifier · Suivre ce point**.
> 📸 **Capture 2** : la vue **« Par responsable »** (onglet Décisions & Actions d'une réunion) + bucket « Sans responsable ».

## 🎯 Objectif
Savoir **ce qui reste à faire**, tous chantiers confondus, et **qui doit le faire**.

## 🕒 Quand l'utiliser
Pour suivre et **relancer**.

## 🔘 Les éléments clés
- **Liste groupée par chantier**, onglets de statut, filtres (corps d'état, ancienneté).
- Sur chaque action : **Clôturer** (avec commentaire + photo → journal), **Planifier** (→ intervention), **Suivre ce point** (rattacher à un point durable).
- **Par responsable** (dans une réunion) — « qui dois-je relancer ? » ; le bucket **Sans responsable** permet d'attribuer en un champ.

## 🧭 Parcours conseillé
`/actions` → filtrer (en retard / par chantier) → agir (clôturer, planifier) ou **Suivre ce point** pour une action orpheline.

## 💡 Conseils
- Une action n'est pas une finalité : c'est *« ce que je dois faire »*. Elle gravite autour d'un **point suivi** (la cause durable).
- « Suivre ce point » : ne nomme pas l'action (« Relancer MOE ») mais le **point durable** derrière (« DOE porte coupe-feu »).

## ⚠️ Erreurs fréquentes
- Confondre **action** et **intervention** : une action n'entre au planning que si elle est **planifiée**.
- Créer un point qui ressemble à une action : l'app vous avertit (⚠).
